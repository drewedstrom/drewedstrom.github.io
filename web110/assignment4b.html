In this activity (activity 1), I examined a tutorial on internet safety. While doing this tutorial, I encountered the Google safe browsing diagnostic tool. To use it, you type the URL of a site you wish to probe into the check site status box and hit enter. If there are any potential concerns, the tool will inform you of them. It’s important to note that typing a website such as twitter.com will not return any meaningful results, as the tool requires you to be more specific than that. I can see this as being a useful point of reference whenever I encounter links to sites that appear to be suspicious. That said, I would likely use this in combination with other tools such as virustotal.com for the most complete picture.

In this activity (activity 4), I spent some time reading up on the various forms of multi-factor authentication. Remarkably, I learned that using SMS for multi-factor authentication is considered ill-advised. This seems to primarily stem from the fact that text messages can be intercepted through an attack known as SIM swapping: where an attacker is able to transfer your phone number to a SIM card in their possession. This enables the attacker to receive texts destined for you—most crucially messages containing two-factor authentication codes. Additionally, there appears to be security issues associated with having various individuals with access to the infrastructure that carries your text messages. Given these vulnerabilities, I am inclined to cease using SMS as a form of MFA and instead ensure that all of my accounts are migrated over to my authenticator app from this point forward.
