In this activity (activity 3), I chose to focus on the ping utility as it was the one I had the least familiarity with. Specifically, I was curious about what arguments can be used with this utility as I have never really found myself using most of them. Though there are a fair few options available, the ones I thought were the most useful were /t and /n. /t is used to continually send echo requests until the user intervenes to stop them while /n is used to specify how many echo requests should be sent. Ultimately, these arguments will enable me to use the ping utility in both a more targeted and thorough manner for troubleshooting networking errors I encounter in both my IT career and my personal life.

In this activity (activity 4), I explored various methods of sharing folders between Windows systems. The revelation that I found to be especially interesting to me was that I could type a UNC path into file explorer to access a shared resource that resides on another system on the network. To do this, I would type a path of the format \\ComputerName\SharedFolder\Resource. Although this would be my preferred method of accessing shared resources as I much prefer a graphical interface, it is also worth mentioning that this can be done with the net use command as well. Where I can see this skill being the most useful is in my IT career, as there are almost certainly going to be networked resources such as shared file servers and print servers present that I will need to be able to access.
